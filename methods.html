Method 1: Survey

The first method is a survey to get a second insight and opinion on the UI to get an idea of which elements are disliked and or need to be altered. I will be asking questions such as:
1. What do you think about the layout of the buttons on the interface?
2. What would you change about the way the controls are displayed?
3. What do you think about the location of the logo?
4. How would you change the layout to make it more simple?
5. What would you change about the controls screen?

Method 2: Heuristic Evaluation

The second method is a Heuristic Evaluation and as such, I will be comparing my UI to make sure they fit these standards which were inspired by Nielsen's Heuristics. Here are the standards that will be considered in review:
1. Can the user see what screen he/she is currently at? The user should be able to clearly identify where he/she currently is within the system i.e. if on the main menu, it should say “Main Menu” somewhere on the screen.
2. Can the user go back to the main menu from any screen within the game?
3. Are the visuals of the Main Menu consistent with other parts of the UI?
4. Can the user use shortcuts to navigate through the menu?
5. Does the system maintain a simplistic design to help new users get established faster?
6. Is the information on the screen relevant to where the user has navigated to?
7. Does the main menu layout accommodate simplicity and ease of access?
8. Does the control screen layout accommodate simplicity and ease of access?
9. Can the average user determine how to access the game and the controls screen in a short amount of time?
